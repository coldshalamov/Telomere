# H179 - Reachable Regime Tax

## Conjecture

Public developmental interpreters are real Telomere-like compression inside the
generated/reachable class, but they do not compress arbitrary data unless the
source restriction is free for some external reason.

## Model

For a public interpreter that maps `G` root/program bits to `P` phenotype bits:

```text
reachable_fraction rho <= 2^(G-P)
source_restriction_tax >= P-G bits
generated_gain ~= P-G-header
```

For `N` phenotype chunks from one root:

```text
raw = N*P
generated = G + header
generated_gain = N*P - G - header
reachable_tax = N*P - G
uniform_net = -header
```

## Results

`H179-reachable_regime_tax.py --root-bits 12,24,64 --phenotype-bits 128 --records 1,8,64 --header-bits 0`

| root bits | chunks | log2 rho | generated gain | reachable tax | uniform net |
| ---: | ---: | ---: | ---: | ---: | ---: |
| 12 | 1 | -116 | 116 | 116 | 0 |
| 12 | 64 | -8180 | 8180 | 8180 | 0 |
| 24 | 64 | -8168 | 8168 | 8168 | 0 |
| 64 | 64 | -8128 | 8128 | 8128 | 0 |

With nonzero headers/framing, the uniform net becomes negative by that overhead.

## Result

Generated/developmental Telomere is a valid stateless decode story for data
known to be generated by the public interpreter. It is not a roughly-all-data
recursive compressor under the uniform hash law. The source tax cancels the
root-vs-phenotype saving.

## Mutation

Keep the developmental/DNA lane as a source-shaped mechanism only if the source
restriction is part of the problem statement. For the arbitrary-content goal,
future work must find a no-tax recurrent population law, not merely a smaller
public generator for a rare reachable subset.
