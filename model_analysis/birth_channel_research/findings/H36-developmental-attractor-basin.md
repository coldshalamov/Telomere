# H36 - Developmental attractor / canalization

Date: 2026-06-17

## Question

Can a DNA-like public developmental system let short genotypes unfold into
large phenotypes while decode stays stateless and recursively fertile?

Mechanism:

```text
record stores genotype / recipe
decoder runs public developmental dynamics
decoder emits phenotype / current layer / future substrate
```

This is the strongest biology-shaped variant: canalized attractors, neutral
basins, regulatory expansion, and genotype-to-phenotype unfolding.

## Support bound

A public decoder with `g` genotype bits can name at most `2^g` phenotypes. For
roughly-all-data coverage of `n` phenotype bits:

```text
2^g >= 2^n
g >= n
```

H36 examples:

```text
n=64,  g=32:  coverage=2^-32, net_if_hit=32 bits
n=64,  g=64:  coverage=1,     net=0 before overhead
n=128, g=64:  coverage=2^-64, net_if_hit=64 bits
n=128, g=128: coverage=1,     net=0 before overhead
```

So a short genotype can name a small phenotype subset. Full arbitrary coverage
costs raw entropy.

## Attractor inverse branch

Attractor dynamics erase information. If an `n`-bit state falls into one of
`2^a` attractors, a lossless arbitrary decoder needs:

```text
attractor id: a bits
inverse branch inside basin: n - a bits
total: n bits
```

H36 examples:

```text
state=64, attractor=8,  residual=0:  coverage=2^-56, cost=8
state=64, attractor=8,  residual=56: coverage=1,     cost=64
state=128,attractor=32, residual=48: coverage=2^-48, cost=80
state=128,attractor=32, residual=96: coverage=1,     cost=128
```

Skipping the inverse branch is lossy for arbitrary states. Storing it returns
to raw size.

## Basin prior

A public basin distribution `Q(x)` is a real source model. It can compress data
drawn from `Q`, but uniform targets pay cross-entropy:

```text
L_Q(x) = -log2 Q(x)
E_{x~Q}[L_Q] = H(Q)
E_{x~U}[L_Q] = n + KL(U || Q) >= n
```

H36 used exact 8-bit toy basin profiles:

```text
random-map-g12: support=256, H(Q)=7.960, source_gain=0.040,
                uniform pure cross-entropy=8.041

zipf-basin-s1.0: support=256, H(Q)=6.222, source_gain=1.778,
                 uniform pure cross-entropy=9.193

zipf-basin-s1.5: support=256, H(Q)=3.916, source_gain=4.084,
                 uniform pure cross-entropy=11.182
```

The stronger the canalization/source bias, the more the matching source
compresses and the more uniform arbitrary targets are penalized.

## Regulatory current+future pairs

A genotype can name current and future observables together:

```text
genotype -> (current phenotype, future substrate)
```

For a source generated by this map, storing the genotype can be a compact
developmental description. For uniform arbitrary pairs, support is still:

```text
coverage = 2^(genotype_bits - current_bits - future_bits)
```

H36 examples:

```text
current=32, future=32, genotype=32: coverage=2^-32, source_gain=32
current=32, future=32, genotype=60: coverage=2^-4,  source_gain=4
current=32, future=32, genotype=64: coverage=1,     source_gain=0
```

## Verdict

Developmental attractors are a real Telomere-shaped route, but they are not a
uniform all-data escape.

What biology appears to buy is not free entropy. It buys a **non-uniform,
canalized source distribution** where a short genotype is the native
description and the expanded phenotype has lower entropy than its raw byte
surface.

For Telomere:

- valid: public developmental source/interpreter, fixed or paid profile id,
  held-out source rows, random/uniform controls negative;
- invalid: using basin support, attractor identity, or regulator choice without
  paying inverse branches/residuals/selectors;
- breakthrough criterion: a source-shaped fertility law must prove
  `gamma > 1` future value per paid latent/regulatory bit.

This is probably the strongest DNA analogy so far: recursive unfolding is
plausible for a public developmental source, not for roughly-all uniform data.
