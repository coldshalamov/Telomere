# Avenue H25 - global referee amortization

Author: Codex continuation from H24. Date: 2026-06-17.
Status: checksum/trial-decode ledger.

## HYPOTHESIS

Maybe the decoder does not need per-record birth/open/status metadata if a
global checksum, end marker, parity lane, or trial-decode referee selects the
right reading after the fact.

This is plausible for small demos: a fixed checksum can distinguish many
candidates. H25 asks whether it scales across many records and arbitrary
passes.

## MECHANISM

Runnable ledger:

- `../H25-global_referee_amortization.py`

If each record leaves `M` candidate readings after structural pruning, then
`R` records leave:

```text
S = M^R
```

joint readings. To select the true one with `safety` bits against false
acceptance, a global referee needs:

```text
C >= R * log2(M) + safety
```

where `C` is checksum/referee width.

For an `E`-bit structural filter over `T` candidate passes:

```text
M = 1 + (T - 1) * 2^-E
```

Singles have `E=0`, so `M=T`.

## RESULT

Run:

```text
python model_analysis\birth_channel_research\H25-global_referee_amortization.py
```

Representative readings:

```text
singles, T=64, E=0:
  bits/record = 6
  64-bit checksum with 32 safety bits covers about 5.33 records

bundle-like, T=64, E=9.36:
  M ~= 1.096
  bits/record ~= 0.132
  64-bit checksum with 32 safety bits covers about 242 records
  at 1000 records, log2 expected false accepts ~= 68

bundle-like, T=65536, E=9.36:
  M ~= 100.96
  bits/record ~= 6.658
  64-bit checksum with 32 safety bits covers about 4.8 records
```

So a checksum can make small finite demonstrations decode. It cannot maintain
arbitrary-pass decode across large files unless it grows with candidate
entropy.

## ACCOUNTING

Stored/paid:

- checksum/referee bits `C`;
- any pass/profile/salt schedule not public-derived.

Derived:

- candidate readings generated by trial decode;
- structural failures that prune wrong candidates.

Hidden if omitted:

- `R * log2(M)` bits of candidate identity when `C` is too small;
- compute explosion if the decoder enumerates `M^R` readings and hopes the
  fixed checksum kills them.

## VERDICT

Global referee bits are useful but finite. They are not a free asymptotic
channel.

The positive version is:

```text
small R or T below the structural knee -> fixed checksum can referee
```

The failing arbitrary-pass version is:

```text
R grows or T exceeds 2^E -> checksum must grow as R * log2(M)
```

This lane therefore supports bounded demos and finite pass windows, not
repeatable/stateless compression maintained over arbitrary passes on large
files.
