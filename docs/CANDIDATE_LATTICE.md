# Candidate Lattice Research

This document defines the experimental superposition/candidate-lattice surface.
It is not the `.tlmr` v2 wire contract.

## Built Today

- `src/superposition.rs` keeps multiple candidates per block during a research
  pass.
- Candidates are deterministically ordered by encoded bit length and seed index.
- Only candidates within an 8-bit delta of the best candidate survive pruning.
- At most three labels, `A`, `B`, and `C`, survive per block after pruning.
- Canonical candidate spans are checked for gap-free, non-overlapping coverage.
- `tests/superposition.rs` covers label promotion, deterministic pruning,
  immediate delta pruning, order independence, and gap/overlap rejection.

## Not Built Into V2

- `.tlmr` v2 does not serialize unresolved candidate alternatives.
- Recursive v2 passes do not branch over superposed candidates.
- Decompression never needs a candidate lattice or index.
- The active indexed/streaming v2 selector receives flat verified candidates and
  emits one selected non-overlapping layer payload.

## Why Keep It

The whitepaper intuition is that a search pass may expose several nearly equal
representations before a later global choice decides which one matters. The
research lattice preserves that intuition without making the file format
ambiguous. It is a laboratory for questions such as:

- Can keeping near-best local alternatives improve selected global savings?
- Does the extra candidate state pay for its memory and search cost?
- Are there deterministic tie rules that beat flat weighted interval selection?
- Can candidate alternatives help future pass alignment without recursive
  branching in the decoder?

## Rules For Future Experiments

- Lattice candidates must be verified by regenerated bytes before use.
- Lattice alternatives may influence compression-time search only.
- Any selected output must collapse to ordinary v1 or v2 records before writing.
- Any recursive experiment must operate on a selected layer output, not on an
  unresolved lattice.
- Telemetry must report candidate count, retained alternatives, selected spans,
  memory use, and final byte delta.
- `docs/SUPERPOSITION_TELEMETRY.md` is the generated selector-audit artifact:
  it proves a deterministic overlap fixture where weighted selection beats
  greedy selection, records retained alternatives, and explains every discarded
  candidate without adding decoder-visible branching.

## Falsification

The lattice research path should be retired if it repeatedly:

- fails to beat the flat weighted selector on planted and structured controls,
- requires memory growth that hides the compression signal,
- creates nondeterministic choices, or
- encourages a decoder-visible branching format.
