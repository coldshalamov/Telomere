Interpretation:

- The span-length sweep isolates the v2 fixed-record overhead threshold: 4-byte spans cannot be emitted profitably, while longer spans increasingly amortize the 5-byte seed-span record.
- The density sweep estimates how much planted/generated material is needed before selected-span savings beat v2 container and literal overhead.
- The memory-scaling sweep keeps planted density fixed at 50% while increasing input size, making throughput, target windows, seed expansions, lookup counts, estimated target-table bytes, and peak working set comparable across sizes.
- The recursive-offset sweep checks whether a first literal layer can create a second-pass block-grid opportunity; this is useful evidence, not a general convergence claim.
- The span-step sweep checks whether sub-block candidate starts can recover an offset generated span in the first pass; it trades a wider search grid for alignment sensitivity.
- The structured-search sweep applies the same streaming engine to generated non-planted JSON under block-step, byte-step, longer-span, and recursive settings; negative results are evidence too.
- The seed-depth sweep isolates a two-byte planted seed that depth 1 cannot find, then measures depth 2 on both that controlled fixture and the structured JSON control.
- Random and binary controls remain in `docs/RESULTS.md`; this sweep intentionally focuses on mechanism curves plus structured-search falsification.
